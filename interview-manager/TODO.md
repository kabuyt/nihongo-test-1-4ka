# 面接事前テスト管理 残タスク

## 重要: 送り出し別アクセス制御の本格化

現在の送り出し別ログインは、静的サイト上で表示を絞る簡易スクリーニング。
スタッフに本格運用で渡す前に、Supabase側のログインとRLSを設定し、DBからも他社データを読めない形にする。

- Supabase AuthでGROP管理者、BARAEN、AKANE、VJCのログインを作る
- `interview_sessions.sender_org` をもとに、送り出しごとの閲覧・更新範囲をRLSで制限する
- GROP管理者だけが面接作成、面接削除、送り出し変更をできるようにする
- 送り出しスタッフは自分の担当面接だけ閲覧・入力できるようにする
- クレペリン結果や候補者情報も、面接の送り出し担当に紐づけて制限する
