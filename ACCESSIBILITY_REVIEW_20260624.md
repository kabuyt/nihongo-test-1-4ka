# Accessibility + Multi-device Review 2026-06-24

対象: `test-preview.html` / `test.html` / `common/renderer.js` / `common/style.css`

範囲: 月間テスト `test1` から `test8`

## 見つけた問題

- 画像に `alt` がない表示経路があり、読み上げ環境で問題画像の意味が伝わりにくかった。
- 記述欄、選択欄、ラジオ系入力の一部にプログラム上の名前がなく、スクリーンリーダーで何の回答欄か分かりにくかった。
- 独自の分類パネルがクリック前提で、キーボードだけでは操作しにくい箇所があった。
- フォーカス表示が弱く、低視力・キーボード操作時に現在位置を追いにくかった。
- スマホ幅で表や画像選択肢が横にはみ出すリスクがあった。
- `maximum-scale=1.0` により、スマホで拡大しにくい設定が残っていた。
- タブ UI と処理状況表示の ARIA が不足していた。

## 修正内容

- 共通レンダラーで、問題画像に文脈ベースの `alt` を自動付与。
- 共通レンダラーで、表示中の入力欄・選択肢・ラジオ/チェック系に `aria-label` を自動付与。
- 分類パネルを `role="button"` / `tabindex="0"` 化し、Enter / Space でクリック相当の操作ができるようにした。
- `test.html` と `test-preview.html` のタブに `tablist` / `tab` / `tabpanel` / `aria-selected` / `aria-controls` を付与。
- 送信・読み込み・採点結果などの状態表示に `role="status"` / `aria-live` を付与。
- スマホ幅の表、画像選択肢、タブ、入力欄にレスポンシブ調整を追加。
- フォーカスリングを高コントラストで見えるように追加。
- `maximum-scale=1.0` を削除し、ユーザー拡大を妨げないようにした。
- ログイン欄、学生情報欄、主要ボタンに不足していたラベルや `type="button"` を追加。

## 検証

- ローカル HTTP サーバーで `test-preview.html` を起動し、管理者ログイン後に `test1` から `test8` を読み込み。
- Chrome headless で 1280px / 768px / 390px の幅を確認。
- 確認結果:
  - `test1` から `test8` すべてで横スクロールなし。
  - 表示中画像の `alt` 欠落なし。
  - 表示中フォーム部品の名前欠落なし。
  - タブの ARIA 状態あり。
  - 分類パネルはフォーカス可能。
- 実行したチェック:
  - `node --check common/renderer.js`
  - `node --check common/grading.js`
  - `node common/grading.test.js` -> 118 tests passed

## 残リスク

- 自動確認は表示・構造・主要キーボード操作中心。実際の音声内容と問題文の一致は、各テスト内容レビューで別途確認が必要。
- test3 以降は問題内容そのものの整備が継続中なので、画像や音声の意味的なズレは今回のアクセシビリティ修正だけでは保証しない。
- スクリーンリーダー実機での読み上げ順は未確認。必要なら NVDA 等での手動確認を追加する。
