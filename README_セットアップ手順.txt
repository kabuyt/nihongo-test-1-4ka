====================================================
みんなの日本語 月間テスト — Web公開セットアップ手順
====================================================

このフォルダの内容をGitHub Pagesで公開し、
受験結果をGoogleスプレッドシートに自動記録する手順です。

所要時間の目安: 約30〜45分

----------------------------------------------------
STEP 1: Googleスプレッドシートを作成する
----------------------------------------------------

1. https://sheets.google.com にアクセスし、新しいスプレッドシートを作成する
2. スプレッドシートに名前をつける（例:「みんなの日本語テスト成績」）
3. スプレッドシートのURLをメモしておく（後でGASと紐づけるため）


----------------------------------------------------
STEP 2: Google Apps Script (GAS) を設定する
----------------------------------------------------

1. スプレッドシートを開いた状態で、メニューから
   「拡張機能」→「Apps Script」をクリック

2. 左側のファイル一覧で「コード.gs」を選択し、
   既存のコードをすべて削除する

3. このフォルダにある「gas_script.js」の内容をすべてコピーし、
   コード.gsに貼り付ける

4. 左上の「保存」ボタン（フロッピーアイコン）をクリック

5. 「デプロイ」→「新しいデプロイ」をクリック

6. 「種類を選択」の歯車アイコンをクリック → 「ウェブアプリ」を選択

7. 以下のように設定する:
   ・説明:「テスト成績受信」（任意）
   ・次のユーザーとして実行:「自分（メールアドレス）」
   ・アクセスできるユーザー:「全員」

8. 「デプロイ」ボタンをクリック

9. Googleアカウントへの許可を求められたら:
   ・「アクセスを承認」→ アカウントを選択
   ・「詳細」→「（プロジェクト名）（安全でないページ）に移動」
   ・「許可」をクリック

10. 「ウェブアプリのURL」が表示される（例: https://script.google.com/macros/s/XXXXX/exec）
    このURLをコピーしておく ←★重要


----------------------------------------------------
STEP 3: index.html にGAS URLを設定する
----------------------------------------------------

1. このフォルダの「index.html」をテキストエディタで開く
   （メモ帳、VS Code、サクラエディタなど何でもOK）

2. ファイルの先頭付近にある以下の行を探す:
   const GAS_URL = 'YOUR_GAS_URL_HERE';

3. 'YOUR_GAS_URL_HERE' の部分を、STEP 2でコピーしたURLに書き換える:
   （例）const GAS_URL = 'https://script.google.com/macros/s/XXXXX/exec';

4. ファイルを保存する


----------------------------------------------------
STEP 4: GitHubにファイルをアップロードする
----------------------------------------------------

【GitHubアカウントをお持ちでない場合】
1. https://github.com にアクセスし、無料アカウントを作成する

【リポジトリの作成】
1. GitHubにログインし、右上の「+」→「New repository」をクリック
2. 以下のように設定:
   ・Repository name: minnano-nihongo-test（任意の名前）
   ・Public（公開）を選択
   ・「Add a README file」にチェックを入れる
3. 「Create repository」をクリック

【ファイルのアップロード】
1. 作成したリポジトリのページで「Add file」→「Upload files」をクリック

2. このフォルダ（Webテスト_公開用）の中にある以下のファイル・フォルダを
   すべてアップロードする:
   ・index.html
   ・audio/（フォルダごと）
   ・images/（フォルダごと）
   ※ gas_script.js と README_セットアップ手順.txt はアップロード不要

3. 「Commit changes」をクリックしてアップロード完了

   ★ audioフォルダ内のMP3ファイルが多い場合:
   GitHubのWebUIでは大量ファイルのアップロードが困難なことがあります。
   その場合はGitHub Desktop（無料アプリ）の使用を推奨します。
   https://desktop.github.com/


----------------------------------------------------
STEP 5: GitHub Pagesを有効にする
----------------------------------------------------

1. リポジトリのページで「Settings」タブをクリック
2. 左メニューの「Pages」をクリック
3. 「Branch」のドロップダウンで「main」を選択し「Save」をクリック
4. しばらく待つと（1〜3分）、以下のようなURLでテストが公開される:
   https://（GitHubユーザー名）.github.io/（リポジトリ名）/

5. このURLをブラウザで開いてテストが表示されれば完成！


----------------------------------------------------
STEP 6: 動作確認
----------------------------------------------------

1. 公開URLをスマートフォンでも開いて、
   テストが正しく表示されるか確認する

2. テストを一通り回答して「採点する」ボタンを押す

3. 「結果を先生に送信する」ボタンを押す

4. Googleスプレッドシートを開き、「成績データ」シートに
   データが追記されていれば設定完了！

   ※ 初回送信時は自動でヘッダー行と「成績データ」シートが作成されます


----------------------------------------------------
注意事項
----------------------------------------------------

・GAS URLは外部に公開しないようにご注意ください
  （URLを知っている人なら誰でもデータを送信できます）

・GitHub Pagesで公開すると、URLを知っている人は誰でもアクセスできます
  受験者以外には共有しないようにしてください

・音声ファイル（MP3）はGitHubの無料プランでは
  1ファイル100MB以内、リポジトリ全体で1GB以内の制限があります

・GASのウェブアプリURLは「再デプロイ」すると変わることがあります
  URLが変わった場合はindex.htmlのGAS_URLも更新してください


----------------------------------------------------
トラブルシューティング
----------------------------------------------------

【テストが表示されない】
→ GitHub PagesのURLが正しいか確認（末尾に/index.htmlは不要）
→ Pagesの有効化から数分待ってから再確認

【音声が再生されない】
→ audioフォルダのMP3ファイルが正しくアップロードされているか確認
→ ファイル名が半角英数字・ハイフンのみか確認（日本語ファイル名は不可）

【結果が送信されない / スプレッドシートに記録されない】
→ index.htmlのGAS_URLが正しく設定されているか確認
→ GASのデプロイ設定で「アクセスできるユーザー:全員」になっているか確認
→ GASエディタで「実行」→「doGet」を手動実行してエラーが出ないか確認

【GAS URLを変えてもう一度デプロイしたい】
→ GASエディタで「デプロイ」→「デプロイを管理」→「編集（鉛筆アイコン）」
→ バージョンを「新しいバージョン」にして「デプロイ」
→ 新しいURLをindex.htmlに貼り直す
